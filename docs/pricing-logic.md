# Pricing Logic: МЕТОД:КЛУБ

Детальное описание логики ценообразования и отображения цен.

---

## Актуальные цены (фиксировано)

### Месячный тариф
```
Старая цена: 9 900 ₽ (зачёркнутая)
Актуальная цена: 5 000 ₽
Стоимость в день: ≈ 167 ₽/день
```

### Годовой тариф
```
Старая цена: 99 900 ₽ (зачёркнутая)
Актуальная цена: 50 000 ₽
Стоимость в день: ≈ 137 ₽/день
```

### Визионеры (премиум)
```
Старая цена: 1 000 000 ₽ (зачёркнутая)
Актуальная цена: 550 000 ₽
Стоимость в день: — (не показываем для этого тарифа)
```

**ВАЖНО:** Эти цены — финальные. Любые другие цифры из старых макетов/версий игнорируются.

## Расчёт "стоимости в день"

### Формула

```
Стоимость в день = Цена тарифа / Количество дней
```

### Месячный тариф

```
5000 ₽ / 30 дней = 166.67 ₽/день ≈ 167 ₽/день
```

### Годовой тариф

```
50000 ₽ / 365 дней = 136.99 ₽/день ≈ 137 ₽/день
```

### Округление

- Округляем до **целого числа** рублей
- Используем стандартное математическое округление (0.5 и выше → вверх)

## Визуальное отображение

### Требования

**1. "Стоимость в день" должна быть заметной**
- ❌ НЕ мелким текстом (14px)
- ✅ Отдельный визуальный блок рядом с ценой
- ✅ Размер шрифта: 18-24px (достаточно крупный)
- ✅ Цвет: золотой акцент (`text-gold-2`)

**2. Формулировка**
```
≈ 167 ₽/день
```
или
```
это всего 167 ₽ в день
```

**3. Расположение**
- Рядом с основной ценой (не в подвале карточки)
- Можно в бейдже/тэге премиального вида

## Психология ценообразования

### Почему "в день" важно

**Проблема:**  
"5000 ₽" звучит как большая сумма, особенно для тех, кто привык к подпискам по 500-1000 ₽.

**Решение:**  
"167 ₽/день" — это **цена одного кофе**. Люди легко сравнивают:
- Кофе в Starbucks ≈ 250-350 ₽
- Обед в кафе ≈ 500-700 ₽
- Такси по городу ≈ 300-500 ₽

"167 ₽/день" — это **меньше** этих привычных трат. Барьер входа снижается.

---

### Почему зачёркнутая цена

**Anchor pricing:**  
Когда человек видит:
```
9 900 ₽  →  5 000 ₽
```

Он воспринимает это как:
- Скидку (выгода)
- Ценность продукта изначально была выше
- Срочность (возможно, цена вернётся)

**Правила:**
- ❌ НЕ добавлять fake urgency ("осталось 2 часа!", "только сегодня!")
- ❌ НЕ врать про скидку, если её нет
- ✅ Просто показываем, что цена была выше, сейчас ниже

## Годовой тариф: особенности

### "6 месяцев + 6 в подарок"

**Формулировка:**
```
Годовой доступ = первые 6 месяцев оплачиваемого доступа + вторые 6 месяцев бонусом
```

**Визуальное выделение:**
- Бейдж "Лучшее предложение" или "Популярный"
- Акцентная граница карточки (золотая)
- Glow-эффект при hover

**Психология:**
- "Бесплатные 6 месяцев" звучат лучше, чем "скидка 50%"
- Усиливает воспринимаемую ценность

## Визионеры: особенности

### Цена без "в день"

Для тарифа за **550 000 ₽** НЕ показываем "стоимость в день", потому что:
- Это премиум-сегмент
- Целевая аудитория — владельцы крупных онлайн-школ с большими оборотами
- Для них 550к — это инвестиция, а не "ежедневная трата"

### Визуальное отличие

Карточка Визионеров должна **выделяться**:
- Более крупная / широкая карточка
- Акцентная граница (золотая, более толстая)
- Glow-эффект
- Бейдж "Премиум-уровень"
- Список бонусов визуально богаче

## Env-переменные для ссылок

### Почему через env

**Гибкость:**
- Можем менять ссылки без правки кода
- Разные ссылки для staging / production
- A/B тестирование (разные ссылки для экспериментов)

### Список переменных

```bash
# .env.local
NEXT_PUBLIC_CLUB_MONTH_URL=https://agkedu.getcourse.ru/club-month
NEXT_PUBLIC_CLUB_YEAR_URL=https://agkedu.getcourse.ru/club-year
NEXT_PUBLIC_CERT_MONTH_URL=https://agkedu.getcourse.ru/cert-month
NEXT_PUBLIC_CERT_YEAR_URL=https://agkedu.getcourse.ru/cert-year
NEXT_PUBLIC_VISIONERS_APPLY_URL=https://agkedu.getcourse.ru/master-groop
```

### Использование

```tsx
const clubYearUrl = process.env.NEXT_PUBLIC_CLUB_YEAR_URL;

<a href={clubYearUrl} target="_blank" rel="noopener noreferrer">
  <Button variant="primary">Оплатить</Button>
</a>
```

**Fallback на случай отсутствия env:**
```tsx
const clubYearUrl = process.env.NEXT_PUBLIC_CLUB_YEAR_URL || '#';

{!clubYearUrl || clubYearUrl === '#' ? (
  <Button variant="primary" disabled>
    Скоро
  </Button>
) : (
  <a href={clubYearUrl}>
    <Button variant="primary">Оплатить</Button>
  </a>
)}
```

## A/B тестирование цен (будущее)

Если захотим тестировать разные цены:

### Вариант 1: Feature flags

```tsx
const isTestGroup = Math.random() > 0.5;

const pricing = isTestGroup
  ? { price: 5000, perDay: 167 }
  : { price: 4500, perDay: 150 };
```

### Вариант 2: Query params

```tsx
const searchParams = useSearchParams();
const variant = searchParams.get('variant'); // ?variant=test

const pricing = variant === 'test'
  ? { price: 4500, perDay: 150 }
  : { price: 5000, perDay: 167 };
```

**Важно:**  
Сейчас A/B тестов нет, но структура кода должна позволять легко добавить это в будущем.

## Форматирование цен

### Локализация

Используем встроенный `toLocaleString()`:

```tsx
const price = 5000;
const formatted = price.toLocaleString('ru-RU'); // "5 000"
```

### С валютой

```tsx
const formatted = price.toLocaleString('ru-RU', {
  style: 'currency',
  currency: 'RUB',
  minimumFractionDigits: 0,
}); // "5 000 ₽"
```

### Утилита (lib/utils.ts)

```tsx
export function formatPrice(price: number): string {
  return price.toLocaleString('ru-RU', {
    style: 'currency',
    currency: 'RUB',
    minimumFractionDigits: 0,
  });
}

// Использование
import { formatPrice } from '@/lib/utils';

<div>{formatPrice(5000)}</div> // "5 000 ₽"
```

```
│  5 000 ₽  (крупно)      │
│  [бейдж: ≈ 167 ₽/день]  │

```

### Mobile

```
┌───────────────────┐
│ 9 900 ₽ (зачёркн) │
│ 5 000 ₽ (крупно)  │
│ [≈ 167 ₽/день]    │
└───────────────────┘
```

На мобильных:
- Цены чуть меньше (но всё равно крупные)
- "В день" остаётся заметным (не убираем!)
